# Life Copilot 产品说明

## 产品定位

Life Copilot 是一个轻量级个人生活助手。第一版只帮助用户把零散想法整理成可执行的任务和计划，并在之后提醒、复盘、积累基础记忆。

第一版不追求“全自动生活管理”，也不替用户执行复杂外部操作。重点是把用户输入变清楚、把计划变具体、把复盘变可持续。

## 第一版范围

第一版只做四件事：

1. 任务解析
2. 计划生成
3. 提醒复盘
4. 基础记忆

## 核心功能

### 1. 任务解析

用户可以输入一句自然语言，例如：

- “我下周要准备体检，帮我安排一下”
- “这周想开始跑步，别太激进”
- “提醒我周五前把报销材料整理好”

系统需要从输入中提取：

- 用户想做什么
- 是否有明确时间
- 是否有截止日期
- 是否需要提醒
- 是否需要拆分成多个步骤

解析结果应当尽量简单、可解释，并允许用户修改。

### 2. 计划生成

系统根据解析出的任务，生成一个基础计划。

计划可以包含：

- 任务标题
- 任务说明
- 建议步骤
- 建议开始时间
- 截止时间
- 提醒时间
- 优先级

第一版计划生成只需要覆盖个人生活场景，不需要处理复杂项目管理、团队协作或跨系统调度。

### 3. 提醒复盘

系统可以在合适时间提醒用户，并在任务结束后进行简单复盘。

提醒内容应包括：

- 当前要做的事
- 为什么现在提醒
- 下一步建议

复盘内容应包括：

- 任务是否完成
- 实际执行情况
- 遇到的问题
- 下次如何调整

复盘的目标不是评价用户，而是帮助系统更了解用户习惯。

### 4. 基础记忆

系统可以保存少量与用户长期偏好相关的信息，用于改进之后的任务解析和计划生成。

基础记忆包括：

- 常用作息偏好
- 常见任务类型
- 用户偏好的提醒方式
- 用户对计划颗粒度的偏好
- 用户明确要求记住的信息

第一版记忆必须保持可查看、可删除、可解释。系统不应隐藏地保存敏感信息。

## 非目标

第一版不做以下事情：

- 不做多 Agent 架构
- 不引入 LangGraph
- 不追求自动化执行一切
- 不连接大量第三方服务
- 不做复杂工作流编排
- 不做团队协作系统
- 不做完整日历替代品
- 不做项目管理软件

## 成功标准

第一版完成后，用户应该可以：

- 输入一段生活事项描述
- 看到系统解析出的任务结构
- 生成一个简单可执行计划
- 收到基础提醒
- 完成后做一次简短复盘
- 让系统记住少量明确偏好

如果这四件事稳定、清楚、可控，第一版就是成功的。
